# 💬 Глобальный чат - Руководство по установке

## Обзор

Добавлена система глобального чата для всех игроков приложения Black Russia Simulator. Чат интегрирован с Supabase и имеет красивый дизайн в стиле социальных сетей.

## Что было добавлено

### 1. **Интерфейс (HTML)**
- **Плавающая кнопка (FAB)** - кружок со смайликом 💬 в нижнем правом углу
- **Модальное окно чата** - полноэкранная модалка с красивым дизайном
- **Значок непрочитанных сообщений** - красный бейдж с числом новых сообщений

### 2. **Стили (CSS)**
- **Стеклянный эффект** (glassmorphism) - современный дизайн
- **Градиенты и тени** - глубина и визуальная иерархия
- **Анимации** - плавные переходы и всплывающие элементы
- **Отзывчивый дизайн** - адаптивен для всех размеров экрана

### 3. **Логика (JavaScript)**
- **ChatSystem** - модуль управления чатом
- Загрузка сообщений из Supabase
- Отправка новых сообщений
- Автоматическое обновление каждые 3 секунды
- Форматирование времени сообщений
- Уникальные аватары для игроков

### 4. **База данных (SQL)**
- Таблица `chat_messages` в Supabase
- Поля:
  - `id` - первичный ключ
  - `player_id` - ID отправителя
  - `player_nick` - никнейм игрока
  - `player_avatar` - emoji-аватар
  - `message` - текст сообщения
  - `created_at` - время создания
  - `updated_at` - время последнего обновления
- Индексы для оптимизации производительности
- Автоматический триггер для обновления `updated_at`

## Установка

### Шаг 1: Создание таблицы в Supabase

1. Откройте Supabase Dashboard вашего проекта
2. Перейдите в **SQL Editor**
3. Создайте новый запрос
4. Скопируйте SQL из файла `supabase_migration_bot.sql` (раздел chat_messages)
5. Выполните запрос

**Или используйте следующий SQL:**

```sql
create table if not exists public.chat_messages (
  id bigserial primary key,
  player_id text not null,
  player_nick text not null,
  player_avatar text default '👤',
  message text not null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists idx_chat_messages_created_at on public.chat_messages(created_at desc);
create index if not exists idx_chat_messages_player_id on public.chat_messages(player_id);

drop trigger if exists trg_chat_messages_updated_at on public.chat_messages;
create trigger trg_chat_messages_updated_at
before update on public.chat_messages
for each row
execute function public.set_updated_at();
```

### Шаг 2: Проверка RLS политик (важно!)

Если у вас включена Row Level Security (RLS), убедитесь что:

1. Политики позволяют **читать** все сообщения (для анонимного ключа):
```sql
create policy "Allow read all chat messages" on public.chat_messages
  for select
  using (true);
```

2. Политики позволяют **писать** сообщения (для анонимного ключа):
```sql
create policy "Allow insert chat messages" on public.chat_messages
  for insert
  with check (true);
```

**Типичная конфигурация:**
- Отключите RLS для `chat_messages` в продакшене или
- Используйте простые политики как выше

### Шаг 3: Проверка подключения

3. Если закрылся на попыне открыться - возможно, нет подключения к БД
4. Проверьте консоль (F12) на ошибки

## Оптимизация Производительности

### Очистка старых сообщений
Раз в месяц запустите в Supabase SQL:

```sql
delete from public.chat_messages 
where created_at < now() - interval '30 days';
```

### Пагинация (для большого количества сообщений)
Если в чате более 1000 сообщений, модифицируйте `loadMessages()`:

```javascript
const { data: msgs, error } = await supabaseClient
  .from('chat_messages')
  .select('*')
  .order('created_at', { ascending: true })
  .range(totalCount - 50, totalCount); // Загрузить последние 50
```

## Фильтрация спама (вариант)

Добавьте проверку на сервере перед сохранением:

```javascript
// В функции sendMessage
if (text.length < 2) {
  alert('Сообщение слишком короткое');
  return;
}

// Защита от флуда (максимум 5 сообщений в минуту)
if (lastMessageTime && Date.now() - lastMessageTime < 12000) {
  alert('Подождите перед следующим сообщением');
  return;
}
lastMessageTime = Date.now();
```

## Решение проблем

### ❌ Чат не открывается
- Проверьте консоль (F12) на ошибки
- Убедитесь, что таблица `chat_messages` создана в Supabase
- Проверьте CORS настройки в Supabase

### ❌ Сообщения не загружаются
- Проверьте RLS политики в Supabase
- Убедитесь, что ваш анонимный ключ имеет доступ к таблице
- Проверьте консоль на ошибки Supabase

### ❌ Не работает отправка сообщений
- Проверьте, что вы авторизованы (должен быть `globalState.currentPlayerId`)
- Убедитесь, что политики позволяют INSERT операции
- Проверьте лимит символов (максимум 256)

### ❌ Медленная загрузка
- Увеличьте интервал обновления (с 3000 до 5000 мс)
- Очистите старые сообщения (старше 30 дней)
- Добавьте пагинацию для больших объемов
